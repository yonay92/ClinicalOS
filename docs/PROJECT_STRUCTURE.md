## Key Architecture Rules

1. `/app/api/**` — HTTP interface only; zero business logic.
2. `/services/**` — Single source of all business logic; one service per domain.
3. `/features/**` — UI concerns; calls server actions or API routes; zero direct Supabase calls.
4. `/lib/supabase/server.ts` — Used by services and API routes only.
5. `/lib/supabase/client.ts` — Used by hooks and client components for realtime/reads.
6. `/lib/supabase/admin.ts` — Used ONLY in Edge Functions and server-controlled service role operations.
7. No component in `/components` or `/features` may contain business logic.
8. Every write in a service must end with an AuditService call.
